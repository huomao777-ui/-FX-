# 项目结构说明

本项目采用按职责和玩法领域分层的目录结构，目标是让运行时系统、玩法模块、界面模块和通用资源彼此独立，方便后续扩展交易、时间、事件、NPC、存档等系统。

## 顶层目录

- `核心/`：全局运行时能力，包含 Autoload 入口和跨玩法系统。
- `玩法/`：游戏世界和核心玩法对象，例如玩家、地图、交易、时间、事件、NPC。
- `界面/`：所有 UI 场景、UI 组件和界面专属美术资源。
- `资源/`：跨模块复用的通用资源，例如字体、通用音效、通用材质。
- `沙盒/`：测试、验证、临时实验场景和脚本，不直接作为正式游戏入口。

## 当前模块

- `核心/自动加载/`：Autoload 场景与入口脚本。
- `核心/系统/`：时间、手机、体力、压力、资产、健康、精神等全局数据系统。
- `玩法/玩家/`：玩家场景、控制脚本、状态机、玩家专属素材和着色器。
- `玩法/地图/站前广场/`：站前广场场景和该地图专属美术资源。
- `界面/组件/`：可复用 UI 组件脚本。
- `界面/场景/开始界面/`：开始界面场景、脚本和专属素材。
- `界面/场景/属性状态/`：属性状态面板场景、脚本和专属素材。
- `界面/场景/手机/`：手机 UI 场景和手机专属素材。
- `界面/场景/外汇应用/`：手机内外汇交易应用相关界面。
- `界面/组件/PhoneStatusBinder.gd`：手机状态栏绑定组件，统一刷新时间、日期、电量、WiFi 和流量。
- `界面/组件/PhoneUsageBinder.gd`：普通手机界面的使用状态绑定组件。
- `资源/字体/`：全项目复用字体。
- `沙盒/测试/`：调试按钮、测试场景、旧实验文件。

## 新增内容放置规则

- 新增 Autoload 或跨系统服务：放入 `核心/自动加载/`，其子系统脚本放入 `核心/系统/` 或新建明确子目录。
- 新增玩法系统：优先放入 `玩法/系统名/`，例如 `玩法/交易/`、`玩法/时间/`、`玩法/NPC/`。
- 新增地图地点：放入 `玩法/地图/地点名/`，地点专属图片放入该目录下的 `美术资源/`。
- 新增 UI 页面：放入 `界面/场景/页面名/`，页面专属图片放入该目录下的 `美术资源/`。
- 新增可复用 UI 控件：放入 `界面/组件/`。
- 新增通用字体、通用音效、跨模块素材：放入 `资源/` 下对应分类。
- 新增临时实验：放入 `沙盒/`，不要放在根目录。

## Godot 编辑注意

- 中文目录名可在 Godot 4.x 正常使用；新增资源后建议等待编辑器完成一次文件系统扫描。
- 移动场景或脚本时，优先在 Godot 编辑器内移动；如在文件系统中移动，需要同步更新 `res://` 引用。
- `.godot/` 下是编辑器缓存，若重构后资源面板短暂显示旧路径，关闭并重新打开项目即可刷新。
